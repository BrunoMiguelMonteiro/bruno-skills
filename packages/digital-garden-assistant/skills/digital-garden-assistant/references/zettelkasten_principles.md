# Zettelkasten Principles

Based on Sönke Ahrens' "How to Take Smart Notes" and Bob Doto's "A System for Writing"

## Core Principles

### 1. Atomicity - One Idea Per Note

Each note should contain exactly one idea, concept, or thought. This enables:
- Maximum reusability
- Clear connections between ideas
- Easier refactoring and reorganization

**Good example:** A note titled "The Collector's Fallacy" explaining only that concept
**Bad example:** A note titled "Note-Taking Methods" covering Zettelkasten, Cornell, and Outline methods

### 2. Autonomy - Notes Should Stand Alone

Each note must be understandable without needing to read other notes. Include enough context so the note makes sense independently, even months or years later.

**Techniques:**
- Write in complete sentences
- Define terms within the note
- Provide necessary context
- Link to related concepts without depending on them

### 3. Connection - Always Link

Notes gain value through connections. Every new note should connect to at least one existing note.

**Connection types:**
- **Direct links:** This concept relates to that concept
- **Hub notes:** Topic overviews that link to specific notes
- **Index notes:** Entry points for exploring themes

### 4. Own Words - Rephrase Everything

Never copy-paste. Always rephrase source material in your own words. This:
- Forces understanding
- Reveals gaps in comprehension
- Makes ideas truly yours
- Improves long-term retention

### 5. Progressive Elaboration

Notes evolve over time:
- Start small (seedling)
- Add connections and context (budding)
- Refine and expand (evergreen)

Don't aim for perfection on first draft. Zettelkasten is a thinking tool, not a publication platform.

## Note Maturity Levels

### Seedling 🌱
- Initial capture of an idea
- Rough, unpolished
- May contain direct quotes (to be rephrased later)
- Few or no connections yet
- Status: `status: seedling`

### Budding 🌿
- Idea rephrased in own words
- Some connections to other notes
- Context added
- Being actively developed
- Status: `status: budding`

### Evergreen 🌲
- Fully developed idea
- Well-connected to other notes
- Clear, concise, autonomous
- Regularly updated and refined
- Publishable quality
- Status: `status: evergreen`

## Writing Process

### Step 1: Capture (Literature Notes)
When reading source material:
- Highlight key ideas
- Write brief notes in margin or separate document
- Always include source citation

### Step 2: Process (Permanent Notes)
Convert literature notes into permanent notes:
- One concept per note
- Rephrase in own words
- Add context for standalone understanding
- Start as seedling

### Step 3: Connect
- Search vault for related concepts
- Add wikilinks `[[Related Concept]]`
- Update related notes to link back
- Consider creating hub/index notes

### Step 4: Elaborate
Over time, return to notes:
- Expand with new insights
- Add examples
- Strengthen connections
- Promote from seedling → budding → evergreen

## Source Material Handling

### From Articles/Web Pages
- Extract 2-5 core concepts
- Create separate note for each concept
- Include source URL in frontmatter
- Add context about why this matters

### From Books
- Create one note per key idea (not per chapter)
- Link related concepts from same book
- Consider creating a hub note for the book
- Include page numbers in backlinks

### From PDFs/Academic Papers
- Focus on novel ideas, not summaries
- Extract methodology, findings, implications separately
- Link to related research
- Note limitations and open questions

### From Videos/Podcasts
- Timestamp key moments
- Extract quotable insights
- Separate concepts from examples
- Link to related written sources

## Common Pitfalls to Avoid

### 1. Collector's Fallacy
Saving content ≠ learning. Only create notes for ideas you actually understand and can explain.

### 2. Premature Taxonomy
Don't organize by topics/folders. Let structure emerge through connections.

### 3. Copy-Paste Syndrome
Copying excerpts creates the illusion of understanding. Always rephrase.

### 4. Orphan Notes
Every note must connect to at least one other note. Isolated notes are lost notes.

### 5. Hub Note Overload
Hub notes are useful but shouldn't become dumping grounds. Keep them focused.

## Quality Indicators

A good permanent note:
- ✓ Contains one clear concept
- ✓ Written in complete sentences
- ✓ Understandable on its own
- ✓ In your own words (no quotes)
- ✓ Connected to related notes
- ✓ Includes source attribution
- ✓ Has actionable wikilinks

A bad permanent note:
- ✗ Multiple unrelated ideas
- ✗ Bullet points without context
- ✗ Copy-pasted excerpts
- ✗ Requires reading other notes to understand
- ✗ No connections (orphan)
- ✗ No source information
